## Mise란?
Mise는 여러 프로그래밍 언어 런타임의 버전을 하나의 도구로 통합 관리하는 CLI입니다. 기존에는 Node.js는 nvm, Python은 pyenv, Go는 gvm, Java는 SDKMAN처럼 언어별로 도구를 나눠 써야 했는데, Mise는 이를 하나의 표준화된 방식으로 통합합니다.

## 기존 도구와 비교했을 때 Mise의 장점
- 통합 설정: 언어별로 따로 관리하던 설정 파일을 `mise.toml` 하나로 통합할 수 있습니다.
- 프로젝트 단위 관리: global과 프로젝트별 버전을 구분해 자동으로 전환합니다.
- 환경 변수까지 통합: 런타임 버전과 환경 변수를 한 파일에서 관리합니다.
- 팀 협업에 유리: `mise.toml`을 저장소에 올리면 동일한 런타임 환경을 쉽게 맞출 수 있습니다.

## 설치 방법
### macOS (Homebrew)
```sh
brew install mise
```

### Windows (Chocolatey)
```sh
choco install mise
```

### Linux / WSL (curl)
```sh
curl https://mise.run | sh
```

### 셸 활성화 설정
#### macOS + zsh (Homebrew 설치 기준)
```sh
echo 'eval "$(mise activate zsh)"' >> ~/.zshrc
```

#### Linux + bash (curl 설치 기준)
```sh
echo 'eval "$(~/.local/bin/mise activate bash)"' >> ~/.bashrc
```

#### Windows
다음 경로를 PATH에 추가합니다.
```
<homedir>\AppData\Local\mise\shims
```

## 설치 가능한 버전 확인
`mise ls-remote`로 설치 가능한 버전을 확인할 수 있습니다.

```sh
mise ls-remote python | tail -3
```
```text
3.13.3
3.13.4
3.13.5
```

```sh
mise ls-remote node | tail -3
```
```text
24.2.0
24.3.0
24.4.1
```

## global 런타임 버전 관리
global에서 런타임을 설정할 때는 `--global` 옵션을 사용합니다.

```sh
mise use -g python@3.13
```
```text
mise python@3.13.5 ✓ installed
mise ~/.config/mise/config.toml tools: python@3.13.5
```

```sh
mise use -g node@24
```
```text
mise node@24.4.1 ✓ installed
mise ~/.config/mise/config.toml tools: node@24.4.1
```

현재 global 설정은 `~/.config/mise/config.toml`에 기록됩니다.

```sh
cat ~/.config/mise/config.toml
```
```toml
[tools]
node = "24"
python = "3.13"
```

현재 활성화된 global 런타임 목록은 다음과 같이 확인합니다.

```sh
mise ls
```
```text
Tool   Version  Source                         Requested
node   24.4.1   ~/.config/mise/config.toml     24
python 3.13.5   ~/.config/mise/config.toml     3.13
```

## 프로젝트별 런타임 버전 관리
프로젝트 루트에서 `mise use`를 실행하면 `mise.toml`에 저장되며, 해당 프로젝트에서만 적용됩니다.

```sh
cd our-project
mise use node@18
```
```text
mise node@18.20.8 ✓ installed
mise ~/work/our-project/mise.toml tools: node@18.20.8
```

버전이 적용되었는지 확인합니다.

```sh
node -v
```
```text
v18.20.8
```

### 프로젝트별 설정 확인

```sh
cat mise.toml
```
```toml
[tools]
node = "18"
```

global과 프로젝트의 런타임이 함께 출력됩니다.

```sh
mise ls
```
```text
Tool   Version  Source                         Requested
node   18.20.8  ~/work/our-project/mise.toml   18
node   24.4.1   ~/.config/mise/config.toml     24
python 3.13.5   ~/.config/mise/config.toml     3.13
```

## 협업 환경에서의 설치

repository에 `mise.toml`을 커밋해 두면, clone 후 `mise install`로 필요한 런타임을 한 번에 설치할 수 있습니다.

```sh
mise install
```
```text
mise python@3.12.11 ✓ installed
mise node@18.20.8 ✓ installed
```

> 주의사항!  
> - `mise use`는 global과 프로젝트 설정을 구분하므로, `-g/--global` 사용 여부를 항상 확인합니다.
> - 팀 협업 시 `mise.toml`을 repository에 포함하고, CI에서도 `mise install`을 사용합니다.
{: .prompt-danger }

## 환경 변수 관리

런타임 버전과 함께 환경 변수도 관리할 수 있습니다.

### 환경변수 등록
`mise set` 명령어로 환경변수를 등록할 수 있습니다.

```sh
mise set DB_USER=root
mise set DB_PASS=1234
```
```text
DB_USER=root
DB_PASS=1234
```

### 현재 등록된 환경 변수 확인
`mise set` 명령어로 현재 설정된 환경변수를 확인할 수 있습니다.

```sh
mise set
```
```text
key     value  source
DB_USER root   ~/work/our-project/mise.toml
DB_PASS 1234   ~/work/our-project/mise.toml
```

### 환경변수 삭제
`mise unset {key}` 명령어로 현재 설정된 환경변수를 삭제할 수 있습니다.

```sh
mise unset DB_PASS
```
```text
removed DB_PASS
```

환경 변수도 `mise.toml`에 같이 저장됩니다.

```toml
[tools]
node = "18"
python = "3.12"

[env]
DB_USER = "root"
DB_PASS = "1234"
```

## 런타임 제거
특정 버전을 더 이상 쓰지 않을 때는 `mise unuse`로 비활성화하고, 필요 시 삭제를 선택합니다.

```sh
mise unuse python@3.12
```
```text
remove core:python@3.12.11 ? Yes No All
```

완전히 삭제하려면 다음 명령을 씁니다.

```sh
mise uninstall python@3.13.5
# 또는
mise rm python@3.13.5
```
```text
mise python@3.13.5 ✓ uninstalled
```

## 런타임 업그레이드
`mise outdated`로 새로운 버전이 있는지 확인합니다.

```sh
mise outdated
```
```text
name   requested  current  latest  source
python 3.13       3.13.3   3.13.5  ~/.config/mise/config.toml
```

`mise upgrade`로 업그레이드를 진행합니다.

```sh
mise upgrade
# 또는
mise up
```
```text
mise python@3.13.5 ✓ installed
mise uninstall python@3.13.3 ✓ remove
```

## 정리

Mise는 언어별 런타임 버전 관리 도구를 하나로 통합해, 설정 파일과 명령 체계를 단순화합니다. global과 프로젝트별 버전을 자동 전환하고, `mise.toml`을 통한 팀 협업도 수월하게 만듭니다. 여러 언어를 동시에 다루는 개발 환경일수록 Mise의 장점이 크게 체감됩니다.
